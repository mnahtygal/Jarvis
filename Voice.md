# Jarvis Voice System

## Purpose
The voice system allows Jarvis to support natural interaction without requiring the dashboard or terminal.

## Current Components
- Microphone input
- Speech-to-text
- Local language model routing
- Text-to-speech
- Optional wake/listen workflow

## Design Principles
- Local-first audio processing
- Low latency where practical
- Clear failure messages
- No cloud dependency required
- Voice should complement the dashboard, not replace it

## Future Work
- Better push-to-talk support
- Wake word refinement
- Speaker output selection
- Conversation mode
- Voice command safety rules
