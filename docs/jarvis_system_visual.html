System Architecture

1. User Input
Input arrives through CLI, API, future UI, voice, or other interface.
2. Brain
core.brain.think() receives and logs the request.
3. Router
core.router.route() decides whether the request is memory, system, time, docs, health, chat, or LLM fallback.
4. Skills
Skills handle specialized tasks such as memory lookup, system status, documentation, or local LLM calls.
5. Storage + Context
PostgreSQL stores memories, conversation history, memory history, session state, and semantic memory structures.
6. Local Inference
llama.cpp serves Qwen3 30B through an OpenAI-compatible local endpoint.
7. Jarvis Response
Jarvis returns a clean response, with Qwen3 thinking blocks stripped from user-visible output.

Starting the Qwen3 Server

cd ~/llama.cpp

./build/bin/llama-server \
  -m ~/models/qwen3-30b/Qwen3-30B-A3B-Q4_K_M.gguf \
  --host 0.0.0.0 \
  --port 8080 \
  -ngl 999 \
  -t $(nproc) \
  --jinja \
  --reasoning-format none

Testing the Local Model API
