# Jarvis Phase 3 – Engineering Assistant

## Goal
Phase 3 evolves Jarvis from a workshop assistant into a broader engineering assistant.

## Focus Areas
- Software engineering support
- Data engineering workflows
- Repository analysis
- Code review assistance
- Project documentation generation
- Engineering decision support
- Local knowledge retrieval
- Workbench-to-code workflows

## Guiding Rule
Phase 3 should only begin after Phase 2 scan, measurement, Maker Lab, and memory workflows are stable.

## Candidate Capabilities
- Analyze code repositories locally
- Explain architecture
- Generate migration plans
- Assist with SQL, Python, React, and Databricks patterns
- Produce technical documentation
- Help with 3D printing and fabrication workflows
- Maintain project memory across sessions

## Success Criteria
Jarvis becomes useful for daily engineering tasks, not just chat or experimentation.
